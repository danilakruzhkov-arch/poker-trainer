# Подключение базы отзывов (Supabase)

Кнопка **«Сообщить об ошибке»** в тренажёре пишет отзывы в базу. Пока база не подключена,
отзывы копятся в браузере автора (localStorage) — их видно в редакторе, блок 0 → «База отзывов».
Чтобы отзывы уходили в облако и были доступны с любого устройства — подними отдельный проект Supabase.

> ВАЖНО: это **отдельный** проект, не трогаем `ClipPoints Supabase` и `ClipPoints Supabase_PROD`.
> poker-trainer держим полностью изолированно.

## Шаги (≈5 минут)

1. **Создать проект.** [app.supabase.com](https://app.supabase.com) → **New project**.
   - Организация: любая (можно та же, что у clippoint — это не смешивает данные, проекты изолированы).
   - Name: `poker-trainer` · Region: `Southeast Asia (Singapore)` (ближе к Бали) · задай пароль БД (можно забыть — он тут не нужен).

2. **Накатить схему.** В проекте → **SQL Editor** → **New query** → вставь весь `supabase-migration.sql` → **Run**.
   Создастся таблица `feedback` с RLS: аноним может только вставлять отзывы, читать — нельзя.

3. **Взять ключи.** **Project Settings → API**:
   - `Project URL` (вида `https://xxxxxxxx.supabase.co`)
   - `Project API keys → anon public` (длинный `eyJ…`). Это ПУБЛИЧНЫЙ ключ, его не страшно светить в браузере.
   - `service_role` НЕ используем на сайте — он только для чтения из дашборда.

4. **Вписать в приложение.** Открой сайт → **Редактор раздач** (пароль) → блок 0 → **«База отзывов · Supabase»**:
   вставь `Project URL` и `anon public key` → **Подключить**. Ключ хранится только в этом браузере.
   Если в очереди уже были отзывы — они улетят в базу сразу.

5. **Проверить.** Вкладка **Тренажёр** → ответь на вопрос → **«Сообщить об ошибке»** → выбери категорию, напиши текст → **Отправить**.
   Затем в Supabase → **Table Editor → feedback** — должна появиться строка.

## Как читать отзывы

- Supabase → **Table Editor → `feedback`** (сортировка по `created_at`), или
- **SQL Editor**:
  ```sql
  select created_at, pack, hand, category, pick_label, hero_label, comment
  from public.feedback order by created_at desc;
  ```

## Что попадает в отзыв

`pack`, `hand`, `street`, номер вопроса, что игрок выбрал (`pick_label` + `pick_grade`),
правильное действие героя (`hero_label`), категория, комментарий, версия приложения, обрезанный user-agent.
Никаких персональных данных — этого достаточно, чтобы найти проблемную раздачу.

## Категории отзывов

| код | смысл |
|---|---|
| `answer`  | неверный правильный ответ или его оценка |
| `hand`    | ошибка в раздаче (карты, размеры, позиции, борд) |
| `video`   | видео/таймкод не совпадает с раздачей |
| `unclear` | вопрос сформулирован непонятно |
| `other`   | другое |

## Заметки по безопасности

- Аноним может только `insert`. Чтение — только из дашборда или service_role. Политики — в миграции.
- Длина комментария ограничена 2000 символами (защита от мусора).
- Если начнётся спам — можно добавить rate-limit (Supabase Edge Function или ограничение по IP через `pg_net`/триггер). Пока не нужно.

## Оффлайн-очередь

Отзывы, оставленные до подключения базы (или когда база временно недоступна), лежат в браузере.
Они сами уходят в базу при следующей загрузке страницы, либо кнопкой **«↑ в базу»** в блоке «База отзывов».
Кнопка **«📋 Отзывы в очереди»** копирует их JSON — на случай, если хочешь забрать вручную.

## Пока база не подключена — всё работает

Кнопка «Сообщить об ошибке» функциональна без Supabase: отзывы сохраняются локально у того, кто открыл сайт.
Supabase нужен только чтобы собирать отзывы централизованно.
