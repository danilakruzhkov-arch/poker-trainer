# База отзывов (Supabase) — подключена

Отзывы из кнопки **«Сообщить об ошибке»** пишутся в Supabase автоматически.
Настраивать в браузере ничего не нужно — адрес проекта и `anon`-ключ **зашиты в сборку** (`index.html`).

- Проект: `mydnywznytluikbwbhsk` (отдельный от clippoint dev/prod)
- Таблица: `public.feedback`
- RLS: аноним может только **вставлять** отзывы; читать/менять/удалять — нельзя

> В сборке только **anon**-ключ (он публичный по дизайну, RLS его ограничивает). `service_role` в коде нет и не будет.

## Как читать отзывы

Supabase → **Table Editor → `feedback`**, либо **SQL Editor**:
```sql
select created_at, pack, hand, category, pick_label, hero_label, comment
from public.feedback order by created_at desc;
```

## Что в каждом отзыве

`pack`, `hand`, `street`, номер вопроса, что игрок выбрал (`pick_label` + `pick_grade`),
правильное действие героя (`hero_label`), категория, комментарий, версия приложения, обрезанный user-agent.
Персональных данных нет — этого хватает, чтобы найти проблемную раздачу.

Категории: `answer` (неверный ответ/оценка) · `hand` (ошибка в раздаче) · `video` (видео/таймкод) · `unclear` (непонятный вопрос) · `other`.

## Оффлайн-очередь

Если у игрока в момент отправки нет сети, отзыв ложится в его браузер и уходит в базу при следующей загрузке страницы.
В редакторе (блок 0 → «Отзывы · Supabase») видно, сколько застряло, и есть кнопки «↑ в базу» / «скопировать».

## Если надо поменять ключ / проект

`anon`-ключ и URL зашиты в `poker-trainer.html` (константы `SUPA_URL` / `SUPA_ANON`), в репозитории собираются в `index.html`.
Поменять проект — правишь эти две константы и передеплой. `service_role` для сайта не нужен никогда.

Ротация `service_role` (если светился): Supabase → **Settings → API → Reset**. На работу сайта не влияет — сайт им не пользуется.

## Схема

Миграция — в [`supabase-migration.sql`](supabase-migration.sql) (уже накачена). Там же политики RLS и грант `insert` для `anon`.

## Хранение раздач — почему не здесь

Раздачи/подборки лежат в **`packs.json` на GitHub** (не в Supabase). Это осознанно:

- packs.json уже версионируется гитом (история правок), бесплатен, читается статикой и кешируется CDN;
- редактор пишет в него через GitHub-токен (Contents API) — уже работает;
- публичное чтение без RLS-возни.

Supabase нужен там, где статический JSON не тянет — **запись от анонимных зрителей** (отзывы).
Если захочешь единый бэкенд (раздачи тоже в Supabase, редактирование без гит-токена, мгновенные апдейты без пересборки Pages) — это отдельная задача, скажи, оценю и сделаю. Пока раздачи в GitHub — правильный инструмент для них.
